# DnD Telegram Bot

Telegram-бот для DnD-кампании: персонажи, опыт, монеты, инвентарь, магазин и предметы с картинками.

Проект сделан как стартовая база, которую можно залить на GitHub, запустить локально или развернуть на VDS через Docker Compose.

---

## Что умеет бот

### Для игрока

- вход по логину и паролю, которые выдал администратор;
- просмотр профиля персонажа;
- просмотр XP и монет;
- просмотр инвентаря;
- просмотр магазина;
- покупка предметов за монеты;
- просмотр карточки предмета с описанием, ценой, редкостью и картинкой.

### Для администратора / мастера

- создание персонажа;
- выдача игроку логина и пароля;
- добавление или списание XP;
- добавление или списание монет;
- создание предметов магазина;
- добавление описания, цены, редкости и картинки предмета;
- выдача предметов персонажу вручную;
- просмотр списка персонажей;
- просмотр списка предметов.

---

## Технологии

- Python 3.12
- aiogram 3
- SQLite
- Docker
- Docker Compose

Почему SQLite: для первой версии DnD-бота этого достаточно. База хранится в одном файле, её легко бэкапить и переносить на VDS. Позже проект можно перевести на PostgreSQL.

## Быстрый старт локально

### 1. Создай бота в Telegram

1. Открой `@BotFather`.
2. Выполни команду `/newbot`.
3. Получи токен бота.

### 2. Узнай свой Telegram ID

Можно использовать бота `@userinfobot`.

Telegram ID нужен, чтобы дать себе права администратора.

### 3. Скопируй `.env.example`

```bash
cp .env.example .env
```

На Windows можно просто скопировать файл `.env.example` и переименовать копию в `.env`.

### 4. Заполни `.env`

```env
BOT_TOKEN=твой_токен_бота
TELEGRAM_ADMIN_IDS=твой_telegram_id
DATABASE_PATH=./data/dnd_bot.sqlite3
CAMPAIGN_NAME=Моя DnD кампания
```

Если администраторов несколько:

```env
TELEGRAM_ADMIN_IDS=123456789,987654321
```

### 5. Создай виртуальное окружение

```bash
python -m venv .venv
```

Активировать на Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
```

Активировать на Linux/macOS:

```bash
source .venv/bin/activate
```

### 6. Установи зависимости

```bash
pip install -r requirements.txt
```

### 7. Запусти бота

```bash
python -m app.main
```

## Запуск через Docker Compose

```bash
docker compose up -d --build
```

Посмотреть логи:

```bash
docker compose logs -f
```

Остановить:

```bash
docker compose down
```

Перезапустить:

```bash
docker compose restart
```

## Развёртывание на VDS

### 1. Установи Docker и Docker Compose

Для Ubuntu/Debian обычно достаточно:

```bash
sudo apt update
sudo apt install -y docker.io docker-compose-plugin git
sudo systemctl enable docker
sudo systemctl start docker
```

### 2. Склонируй репозиторий

```bash
git clone https://github.com/USERNAME/dnd-telegram-bot.git
cd dnd-telegram-bot
```

### 3. Создай `.env`

```bash
cp .env.example .env
nano .env
```

Заполни:

```env
BOT_TOKEN=твой_токен_бота
TELEGRAM_ADMIN_IDS=твой_telegram_id
DATABASE_PATH=/app/data/dnd_bot.sqlite3
CAMPAIGN_NAME=Моя DnD кампания
```

### 4. Запусти контейнер

```bash
docker compose up -d --build
```

### 5. Проверь логи

```bash
docker compose logs -f
```

Если всё хорошо, в логах будет сообщение:

```text
Bot started in long polling mode
```

## Как пользоваться

### Первый вход администратора

1. Открой своего Telegram-бота.
2. Напиши `/start`.
3. Нажми `Админ-панель` или напиши `/admin`.

Кнопка админ-панели появится только у пользователей, чей Telegram ID указан в `.env`.

---

## Основной сценарий мастера

### 1. Создать персонажа

Админ-панель → `Создать персонажа`

Бот попросит:

1. логин;
2. пароль;
3. имя персонажа.

После создания бот покажет логин и пароль. Их нужно передать игроку.

Пароль в базе хранится не открытым текстом, а в виде хэша.

### 2. Игрок входит в персонажа

Игрок пишет:

```text
/login
```

Дальше вводит логин и пароль.

После входа Telegram-аккаунт игрока привязывается к персонажу.

### 3. Создать предмет

Админ-панель → `Создать предмет`

Бот попросит:

1. название;
2. описание;
3. цену;
4. редкость;
5. картинку.

Картинку нужно отправить как обычное фото в Telegram. Бот сохранит `file_id` картинки, поэтому сам файл отдельно хранить не нужно.

### 4. Выдать опыт или монеты

Админ-панель → `Добавить XP` или `Добавить монеты`

Можно вводить положительное или отрицательное число.

Примеры:

```text
50
```

```text
-10
```

### 5. Выдать предмет вручную

Админ-панель → `Выдать предмет`

Можно выдать предмет игроку или забрать предмет отрицательным количеством.

## Команды бота

```text
/start      — запустить бота
/menu       — главное меню
/login      — войти в персонажа
/logout     — выйти из персонажа
/profile    — профиль персонажа
/inventory  — инвентарь
/shop       — магазин
/admin      — админ-панель
/cancel     — отменить текущее действие
```

## База данных

Бот сам создаёт SQLite-базу при первом запуске.

По умолчанию база хранится здесь:

```text
./data/dnd_bot.sqlite3
```

В Docker база хранится в volume-папке проекта:

```text
./data
```

Эта папка добавлена в `.gitignore`, чтобы случайно не залить базу с данными игроков на GitHub.

---

## Таблицы

### `characters`

Хранит персонажей:

- логин;
- хэш пароля;
- Telegram ID игрока;
- имя персонажа;
- XP;
- монеты.

### `shop_items`

Хранит предметы:

- название;
- описание;
- цену;
- редкость;
- Telegram `file_id` картинки;
- активность предмета в магазине.

### `inventory`

Хранит связь персонажей и предметов:

- какой персонаж;
- какой предмет;
- количество.

### `transactions`

Хранит историю операций:

- выдача XP;
- выдача монет;
- выдача предметов;
- покупка предметов.

## Важные замечания по безопасности

1. Никогда не пушь `.env` на GitHub.
2. Никогда не публикуй токен Telegram-бота.
3. Не добавляй файл базы данных в репозиторий.
4. Если токен случайно попал в GitHub — сразу перевыпусти токен через `@BotFather`.
5. Пароли игроков хранятся в виде хэша, но всё равно лучше не использовать настоящие личные пароли.
